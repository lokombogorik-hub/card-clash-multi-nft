# Card Clash — что починено и что нужно для запуска

Все правки сделаны **только на бэкенде** (`backend/`). Игровой фронтенд не трогали — он
уже умеет обрабатывать нужные WebSocket-сообщения (`get_state`, `player_disconnected`,
`game_over`, `escrow_locked`).

## Что исправлено

**1. Лок NFT (эскроу) — единый источник правды.**
Раньше `register_deposits` и `confirm_escrow` спорили за один объект матча (плюс
`confirm_escrow` дублировал депозиты в БД), отсюда «лок второго игрока не виден» и
зависания. Теперь:
- единый пересчёт `recompute_escrow_lock` (залочено, когда оба подтвердили — по флагу
  ИЛИ по ≥5 депозитам);
- пер-матчевый `asyncio.Lock` против гонок;
- депозиты пишутся идемпотентно (без дублей, иначе ломался claim);
- по факту лока шлётся `escrow_locked` по WS и игра стартует автоматически
  (`try_start_game`).

**2. Очки и рейтинг считаются на сервере.**
Раньше WS первым ставил `status=finished`, и клиентский `/finish` выходил по
`already_finished` → рейтинг не начислялся вообще. Теперь рейтинг начисляется
авторитетно в WS `game_over` (идемпотентно, флаг `rating_applied`), а `/finish`
больше не доверяет `winner` от клиента (проигравший не объявит победителем себя).

**3. Матчмейкинг по реальному рангу.**
ELO больше не захардкожен `1000` — берётся реальный `elo_rating` из БД. Добавлена
защита от дублей комнат (если есть незавершённый матч — возвращаем его). Колода для
матча грузится из БД, а не только из памяти (без подмены на чужой `default_user`).

**4. Реконнект и дисконнект.**
- добавлен обработчик `get_state` — состояние возвращается вернувшемуся игроку;
- при коннекте в идущую партию состояние отдаётся сразу;
- при вылете оппоненту шлётся `player_disconnected` с дедлайном, запускается таймер;
- если игрок не вернулся за `RECONNECT_TIMEOUT_SECONDS` (180с) — **поражение
  вышедшему** (форфейт): оставшийся побеждает, получает рейтинг и право забрать NFT.

**5. Безопасность/конфиг.**
- CORS: убрана невалидная пара `allow_origins=["*"] + allow_credentials=True`
  (теперь `credentials=False`, домены можно сузить через `ALLOWED_ORIGINS`);
- токен Telegram-бота вынесен из кода в `TELEGRAM_BOT_TOKEN`.

## ⚠️ Срочно: отозвать токен бота

Старый токен бота лежал прямо в `backend/tg/bot.py` и теперь считается
скомпрометированным. В @BotFather: `/revoke` → выпусти новый → положи в env
(`TELEGRAM_BOT_TOKEN`). Старый удали отовсюду.

## Переменные окружения

### Backend
| Переменная | Зачем | Обязательно |
|---|---|---|
| `DATABASE_URL` | PostgreSQL (`postgresql://...`) | да |
| `JWT_SECRET` | подпись токенов | да |
| `TELEGRAM_BOT_TOKEN` | проверка initData + бот | да |
| `ESCROW_WALLET` | NEAR-аккаунт эскроу | да (для PvP на ставку) |
| `ESCROW_PRIVATE_KEY` | ключ эскроу для выплат/возвратов NFT | да (для PvP на ставку) |
| `NFT_CONTRACT_ID` | контракт NFT-карт | да (для PvP на ставку) |
| `NEAR_RPC_URL` | по умолчанию mainnet | нет |
| `ALLOWED_ORIGINS` | список доменов через запятую (по умолчанию `*`) | нет |

Без `ESCROW_PRIVATE_KEY` + `NFT_CONTRACT_ID` лок/возврат NFT работать не будет —
именно это даёт симптом «ошибка кошелька/конфига».

### Frontend (Vite, на Vercel)
| Переменная | Зачем |
|---|---|
| `VITE_API_BASE_URL` (или `VITE_API_URL`) | адрес API (для REST и WebSocket) |
| `VITE_NEAR_ESCROW_CONTRACT_ID` | аккаунт эскроу (если не задан — PvP идёт в beta без ставок) |
| `VITE_NEAR_NFT_CONTRACT_ID` | контракт NFT |

## Рекомендую проверить перед массовым запуском
- Прогнать матч двух реальных Telegram-аккаунтов: лок обоих → старт → партия → claim.
- Проверить реконнект: свернуть/обновить WebApp в середине партии — игра должна
  восстановиться. Полный выход одного на >3 мин — второму засчитывается победа.
- Убедиться, что у обоих игроков колода берётся из их NFT (а не случайная) —
  это значит, что колода реально сохранена в БД (`/api/decks/save`).
